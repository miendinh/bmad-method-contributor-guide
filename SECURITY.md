# Security Policy

> ⚠️ **This is for THIS DOCUMENTATION repository only.**
> For BMAD-METHOD framework security: <https://github.com/bmad-code-org/BMAD-METHOD/blob/main/SECURITY.md>

---

## Reporting security issues

### Issues IN this documentation

If you find:
- **Incorrect security information** that could mislead users
- **Outdated security advice** applicable to BMAD-METHOD
- **Links to malicious resources** accidentally included
- **Leaked credentials** accidentally committed

Report via:
- Open a **private issue** in this repository (if supported)
- Or contact maintainer via the contact method in README

**Do NOT publicly disclose** security issues until fixed.

### Issues IN BMAD-METHOD framework

**Report directly to BMad Code, LLC:**
- Follow: <https://github.com/bmad-code-org/BMAD-METHOD/blob/main/SECURITY.md>
- **Do not report framework vulnerabilities in this repository**

## Scope of this security policy

### In scope

- Content accuracy of security-related information in this documentation
- Any credentials/tokens accidentally committed
- Malicious links or recommendations in this documentation

### Out of scope

- Security of BMAD-METHOD framework itself (not our domain)
- Security of tools mentioned (Claude Code, Cursor, etc.)
- Third-party dependencies of BMad

## Response timeline

| Severity | Response time |
|----------|---------------|
| Critical (active exploitation) | 24 hours |
| High (potential harm) | 3 days |
| Medium (misleading info) | 1 week |
| Low (typo, minor) | Best effort |

## Acknowledgments

Security researchers who report issues responsibly are acknowledged (with permission) in release notes.
